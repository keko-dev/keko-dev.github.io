新手入门运维：服务器基础管理与核心实操技巧

很多新手开发者在搭建完个人博客（如Hexo）、部署完小型项目后，会陷入一个误区：忽略运维环节，认为“部署完成就万事大吉”。直到服务器卡顿、服务宕机、数据丢失，才发现运维的重要性——运维的核心不是“修bug”，而是“防患于未然”，用简单的操作，最大化保障服务器和服务的稳定运行。

本文针对新手运维，摒弃复杂的理论，聚焦Linux服务器（阿里云、腾讯云等主流云服务器通用）的核心运维场景，从「服务器基础配置」「日常运维实操」「应急问题排查」三个维度，分享可直接照搬的操作方法，帮你快速掌握基础运维能力，守住自己的服务器“阵地”。

一、新手必做：服务器初始化配置（部署前必看）

无论是新购买的云服务器，还是刚重置系统的服务器，初始化配置是运维的第一步，能避免后续很多安全隐患和操作麻烦，重点做好以下4件事。

  1. 重置密码，创建普通用户（安全第一）

新服务器默认密码复杂且不易记忆，同时建议创建普通用户（避免长期使用root用户操作，降低误操作风险）：

1. 重置root用户密码（登录服务器后执行）

passwd root

输入新密码（输入时不显示，确认时再次输入），密码建议包含字母、数字、特殊符号

2. 创建普通用户（示例用户名：ops，可自定义）

useradd ops

给普通用户设置密码

passwd ops

3. 给普通用户授权（允许执行sudo命令，临时获取root权限）

usermod -aG wheel ops

后续操作建议使用普通用户登录，需要执行高权限命令时，在命令前加 sudo 即可（如 sudo systemctl restart nginx）。

  1. 关闭无用端口，开放核心端口（安全防护）

服务器默认开放很多端口，无用端口会增加安全隐患，只需开放核心端口即可（个人博客常用端口：22-SSH登录、80-HTTP访问、443-HTTPS访问）：

1. 查看当前开放的端口（CentOS系统）

firewall-cmd –list-ports

2. 关闭防火墙（新手推荐，简化操作，后续可按需开启）

systemctl stop firewalld
systemctl disable firewalld

3. 云服务器安全组配置（关键！）

登录云服务器控制台（阿里云/腾讯云）→ 找到「安全组」→ 入方向规则

只保留以下3个核心端口，删除其他无用规则：

- 22端口：允许SSH登录（可限制仅自己的IP访问，更安全）

- 80端口：允许HTTP访问（博客正常访问）

- 443端口：允许HTTPS访问（可选，配置SSL后需要）

注意：关闭防火墙仅适合新手入门和个人博客场景，若部署生产项目，建议开启防火墙并精准配置端口规则。

  1. 更新系统软件，安装核心工具（提升稳定性）

新服务器系统软件可能存在更新不及时的情况，更新软件并安装常用运维工具，能提升服务器稳定性和操作效率：

CentOS系统更新软件（yum包管理器）

sudo yum update -y

Ubuntu系统更新软件（apt包管理器）

sudo apt update && sudo apt upgrade -y

安装常用运维工具（无论CentOS还是Ubuntu，按需安装）

sudo yum install -y wget net-tools htop vim # CentOS
sudo apt install -y wget net-tools htop vim # Ubuntu

工具说明：

wget：下载文件；net-tools：查看端口、网络状态；htop：进程管理；vim：文本编辑

  1. 配置时间同步（避免时间错乱）

服务器时间错乱会导致日志记录异常、定时任务执行失败（如自动备份），配置时间同步能解决该问题：

安装时间同步工具（CentOS）

sudo yum install -y chrony

启动并设置开机自启

sudo systemctl start chronyd
sudo systemctl enable chronyd

查看时间同步状态（确认同步成功）

timedatectl status

正常提示：System clock synchronized: yes（已同步）

二、日常运维实操：每天5分钟，保障服务器稳定

日常运维无需复杂操作，每天花5分钟完成基础巡检，就能避免90%的服务器问题。重点关注「资源占用」「服务状态」「日志情况」三个核心维度，操作命令整理如下，可直接复制执行。

  1. 资源占用巡检（CPU、内存、磁盘）

CPU、内存、磁盘是服务器的核心资源，占用过高会导致服务卡顿、宕机，每天巡检时重点查看：

1. 查看CPU占用情况（实时排序，按P键刷新）

htop

2. 查看内存使用情况（直观显示可用内存）

free -h

3. 查看磁盘空间占用（重点关注使用率）

df -h

4. 查看磁盘大文件（排查异常占用，比如日志过大）

du -sh /var/* | sort -rh | head -5

关键判断标准（新手可直接参考）：

  • CPU：单个进程占用不超过50%，整体负载不超过CPU核心数；

  • 内存：可用内存不低于1GiB，若过低，清理缓存（sudo sync && echo 3 > /proc/sys/vm/drop_caches）；

  • 磁盘：分区使用率不超过80%，超过后及时清理无用文件（旧日志、过期备份）。
